| 1.5 | `(int)3.9`=3(버림!), `(double)7/2`=3.5, int 범위: 약 +-21억 |

| 1.15 | String 불변, `substring(from,to)` to 미포함, `indexOf` 없으면 -1 |

### 필수 코드 패턴

```java
// 랜덤 정수 (min ~ max 포함)
int r = (int)(Math.random() * (max - min + 1)) + min;

// 캐스팅
int a = (int) 3.9;           // 3 (버림, 반올림 아님!)
double d = (double) 7 / 2;   // 3.5 (하나라도 double이면 double)

// String 핵심 메서드
str.length()              // 길이
str.substring(a, b)       // 인덱스 a ~ b-1
str.substring(a)          // 인덱스 a ~ 끝
str.indexOf("x")          // 처음 위치, 없으면 -1
str.equals(other)         // 내용 비교 (== 쓰면 안 됨!)
str.compareTo(other)      // 사전순: 음수(앞) 0(같음) 양수(뒤)
```

### 함정 체크리스트

- [ ] `7/2 = 3` (3.5 아님! int/int=int)
- [ ] `(int)3.9 = 3` (반올림 아님, 버림!)
- [ ] `substring(1,3)` -> 인덱스 1,2만 (3 미포함)
- [ ] String 비교는 `.equals()` (`==`는 주소 비교!)
- [ ] `"a"+1+2 = "a12"`, `1+2+"a" = "3a"` (왼->오 결합)
- [ ] `Math.pow()` 반환은 항상 `double` -> int에 직접 대입 불가
- [ ] `(int)Math.random()*10` = 항상 0! -> `(int)(Math.random()*10)` 괄호 주의
- [ ] `Math.random()` 범위: `[0.0, 1.0)` -- 1.0 미포함

### Quick Reference 연결

`String`: `length()` `substring()` `indexOf()` `equals()` `compareTo()` `split()`
`Math`: `abs()` `pow()` `sqrt()` `random()`
`Integer`: `MIN_VALUE` `MAX_VALUE` `parseInt()`

## Unit 2: Selection and Iteration (25-35%)

### 한 줄 요약

| 토픽 | 핵심 |
|------|------|
| 2.1 | 모든 프로그램 = 순차 + 선택(if) + 반복(loop) |
| 2.2 | 관계연산자 `== != < > <= >=`, boolean은 primitive |
| 2.3 | `if` / `if-else` / `if-else if-else`, 중괄호 없으면 한 문장만 |
| 2.4 | 중첩 if: 안쪽 if는 바깥 조건이 true일 때만 실행 |
| 2.5 | `&&`(AND) `||`(OR) `!`(NOT), 단축평가: &&앞false->뒤안봄, ||앞true->뒤안봄 |
| 2.6 | De Morgan: `!(A&&B)` = `!A||!B`, `!(A||B)` = `!A&&!B` |
| 2.7 | `while`: 조건 먼저 확인, 무한루프 주의, 0회 실행 가능 |
| 2.8 | `for(init; cond; update)`: init 1회->cond확인->실행->update->cond... |
| 2.9 | 최대/최소 찾기, 합계/평균, 카운팅 패턴 |
| 2.10 | String 순회: `for(int i=0; i<s.length(); i++)` + `substring(i,i+1)` |
| 2.11 | 중첩 루프: 바깥 1회당 안쪽 전체 실행, 시간복잡도 O(n^2) |
| 2.12 | 비공식 실행시간: 단일루프 O(n), 중첩루프 O(n^2), 이진탐색 O(log n) |

### 필수 코드 패턴

```java
// 최대값 찾기
int max = Integer.MIN_VALUE;
for (int i = 0; i < arr.length; i++) {
    if (arr[i] > max) max = arr[i];
}

// String 문자별 순회
for (int i = 0; i < s.length(); i++) {
    String ch = s.substring(i, i + 1);
}

// 짝수/홀수 카운팅
int count = 0;
for (int i = 0; i < arr.length; i++) {
    if (arr[i] % 2 == 0) count++;
}

// De Morgan 변환
!(a && b)  ==  !a || !b
!(a || b)  ==  !a && !b
!(a > b)   ==  a <= b
!(a >= b)  ==  a < b
```

### 함정 체크리스트

- [ ] `if(x = 5)` 컴파일 에러! (비교는 `==`)
- [ ] `==`로 double 비교 위험 -> `Math.abs(a-b) < 0.0001`
- [ ] `for(int i=0; i<5; i++)` -> 0,1,2,3,4 (5번), `<=5`면 6번!
- [ ] `while` 무한루프: 조건 변수를 루프 안에서 변경 안 하면 무한
- [ ] 단축평가: `if(arr != null && arr.length > 0)` 순서 바꾸면 NPE!
- [ ] De Morgan 변환 시 `&&` <-> `||` 바꾸는 것 잊지 말기
- [ ] enhanced for `for(int x : arr)` 에서 x 수정해도 원본 불변
- [ ] 중첩 if의 else는 가장 가까운 if에 매칭

### Quick Reference 연결

`Integer.MIN_VALUE` / `Integer.MAX_VALUE` (최대/최소 초기화용)

## Unit 3: Class Creation (10-18%)

### 한 줄 요약

| 토픽 | 핵심 |
|------|------|
| 3.1 | 추상화: Data(이름으로 사용) vs Procedural(메서드 이름으로 사용) |
| 3.2 | 프로그램의 사회적 영향, reliability, open source vs proprietary |
| 3.3 | 캡슐화: instance variable은 `private`, constructor/method는 `public` |
| 3.4 | Constructor: 반환 타입 없음, 이름=클래스명, 기본값 0/0.0/false/null |
| 3.5 | accessor(getter, non-void) vs mutator(setter, void), pass by value |
| 3.6 | 참조형 매개변수: 상태변경->원본변경, 참조재할당->원본불변 |
| 3.7 | `static`=클래스소속, instance 접근 불가, `static final`=상수 |
| 3.8 | scope: local variable은 블록 안에서만, `public/private` 불가 |
| 3.9 | `this`=현재객체, shadowing 해결, static에서 사용 불가 |

### 필수 코드 패턴

```java
// 완전한 클래스 템플릿 (FRQ Q2)
public class ClassName {
    // 1. private instance variables
    private String name;
    private int value;

    // 2. Constructor (public, 반환타입 없음)
    public ClassName(String name, int value) {
        this.name = name;
        this.value = value;
    }

    // 3. Accessor (getter)
    public String getName() { return name; }

    // 4. Mutator (setter)
    public void setValue(int value) { this.value = value; }

    // 5. 표시용 메서드 (명시적 이름 — toString 오버라이드는 CED 1.15 EXCLUSION)
    public String getInfo() {
        return name + ": " + value;
    }
}
// 사용: System.out.println(obj.getInfo());  ← println(obj)로 자동 호출되지 않음 (의도된 명시 호출)
```

```java
// static 변수로 객체 수 추적
private static int count = 0;
public ClassName(...) { count++; }
public static int getCount() { return count; }

// 방어적 복사 (mutable 참조)
public ClassName(ArrayList<String> list) {
    this.list = new ArrayList<>(list); // 복사본 저장
}
```

### 함정 체크리스트

- [ ] instance variable은 반드시 `private` (public이면 캡슐화 위반 감점)
- [ ] constructor에 `void` 쓰면 메서드가 됨 (constructor 아님!)
- [ ] 생성자 하나라도 정의하면 default constructor 자동 소멸
- [ ] 기본값: `int`->0, `double`->0.0, `boolean`->false, 참조->null
- [ ] `static` 메서드에서 `this`, instance variable 접근 불가
- [ ] local variable에는 `public`/`private` 사용 불가
- [ ] primitive 매개변수 = 복사본 (원본 불변), 참조형 = 같은 객체 (상태변경 영향)
- [ ] `String`은 immutable -> 참조형이지만 변경 불가 (예외!)
- [ ] non-void 메서드는 모든 경로에서 `return` 필수

### Quick Reference 연결

`Object.equals()` `Object.toString()` (String 등은 값 비교/문자열 표현 제공). ⚠️ **호출은 가능**(트레이싱 IN), 학생이 **오버라이드 작성은 금지**(CED 1.15, 2.6 EXCLUSION) — 표시용 메서드는 `getInfo()` 등 명시적 이름으로 정의.

### 필수 코드 패턴

```java
// 배열 순회 (기본)
for (int i = 0; i < arr.length; i++) {
    // arr[i] 사용
}
// enhanced for (읽기 전용)
for (int val : arr) {
    // val 사용 (수정 불가!)
}

// ArrayList 삭제 (뒤에서부터 -- 가장 안전)
for (int i = list.size() - 1; i >= 0; i--) {
    if (조건) list.remove(i);
}
// ArrayList 삭제 (앞에서부터 -- i 보정)
for (int i = 0; i < list.size(); i++) {
    if (조건) { list.remove(i); i--; }
}

// 2D 배열 row-major 순회 (FRQ Q4)
for (int r = 0; r < arr.length; r++) {
    for (int c = 0; c < arr[0].length; c++) {
        // arr[r][c] 사용
    }
}

// 이진 탐색 (정렬된 배열 전제)
int lo = 0, hi = arr.length - 1;
while (lo <= hi) {
    int mid = (lo + hi) / 2;
    if (arr[mid] == target) return mid;
    else if (arr[mid] < target) lo = mid + 1;
    else hi = mid - 1;
}
return -1; // 못 찾음

// Selection Sort
for (int i = 0; i < arr.length - 1; i++) {
    int minIdx = i;
    for (int j = i + 1; j < arr.length; j++) {
        if (arr[j] < arr[minIdx]) minIdx = j;
    }
    int temp = arr[i]; arr[i] = arr[minIdx]; arr[minIdx] = temp;
}

// Insertion Sort
for (int i = 1; i < arr.length; i++) {
    int key = arr[i];
    int j = i - 1;
    while (j >= 0 && arr[j] > key) {
        arr[j + 1] = arr[j];
        j--;
    }
    arr[j + 1] = key;
}

// File/Scanner 파일 읽기 (2025-26 신규!)
// 반드시 throws IOException 선언 필요
public static void readFile(String path) throws IOException {
    Scanner sc = new Scanner(new File(path));
    while (sc.hasNext()) {
        String line = sc.nextLine();
        // line 처리
    }
}
// nextInt() 후 nextLine() 주의 (시험 범위 밖 — CED EXCLUSION):
// nextInt()는 개행문자를 남김. 이 혼용 분석은 AP 시험에 출제되지 않음.
// sc.nextInt(); sc.nextLine(); // 빈 줄 소비 → sc.nextLine(); // 실제 데이터

// 재귀 기본 구조 (MCQ 트레이싱 전용 — 작성은 시험 범위 밖)
public int factorial(int n) {
    if (n <= 1) return 1;          // base case
    return n * factorial(n - 1);   // recursive case
}
```

### 함정 체크리스트

- [ ] 배열은 `.length` (괄호없음), String은 `.length()`, ArrayList는 `.size()`
- [ ] 배열 `arr[i]`, ArrayList `list.get(i)` -- 대괄호 vs 메서드
- [ ] 배열은 크기 고정 (생성 후 변경 불가), ArrayList는 가변
- [ ] `ArrayList<int>` 불가! -> `ArrayList<Integer>` (Wrapper 필수)
- [ ] `list.remove(5)` = **인덱스 5의 원소** 삭제 (Quick Reference의 `remove`는 `remove(int index)` 한 가지뿐. "값 X 삭제"는 직접 검색 루프로 인덱스를 찾아 `remove(int)` 호출)
- [ ] ArrayList 앞에서 삭제하면 뒤 요소 인덱스 1씩 감소 -> 건너뜀 버그!
- [ ] enhanced for 안에서 ArrayList `.remove()` 하면 `ConcurrentModificationException`
- [ ] 2D 배열: `arr.length`=행 수, `arr[0].length`=열 수 (헷갈림 주의)
- [ ] 이진탐색은 **정렬된 배열에서만** 사용 가능
- [ ] 재귀: base case 빠지면 `StackOverflowError`
- [ ] `set()`은 이전 값 반환, `remove()`도 삭제된 값 반환

### Quick Reference 연결

`ArrayList`: `size()` `add(obj)` `add(idx,obj)` `get(idx)` `set(idx,obj)` `remove(idx)`
`Scanner`: `nextInt()` `nextDouble()` `nextLine()` `next()` `hasNext()` `close()`
