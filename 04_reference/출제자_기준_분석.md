| 1 | Design Code | 2-10% |

**핵심 인사이트:** MC의 절반 이상이 Practice 3 = "주어진 코드의 출력/동작을 분석"하는 능력. 코드를 쓰는 것보다 **읽고 추적하는 능력**이 MC에서 더 중요.

## 4. 채점 기준에서 드러나는 출제자의 철학

### 4-1. Algorithm Point vs Non-Algorithm Point

채점 기준의 각 포인트는 두 종류:
- **Algorithm point**: 전체 로직이 올바르게 조합되어야 득점. 부분 오류 시 0점.
- **Non-algorithm point**: 개별 단계만 맞으면 다른 오류가 있어도 득점 가능.

**출제 의도:** "전체를 조합하는 능력"과 "개별 단계를 아는 능력"을 분리 평가.

### 4-2. 감점하지 않는 것 (Errors to Ignore) — 매우 중요!

출제자가 **신경 쓰지 않는** 사소한 실수:
- `;` 누락 (의도가 명확하면)
- `{}` 누락 (들여쓰기로 의도가 보이면)
- `=` 대신 `==`, 또는 반대
- `length` vs `size` vs `length()` 혼용
- `[i,j]` 대신 `[i][j]` (또는 반대)
- `private`/`public` 키워드를 지역변수에 붙여도
- 배열 선언 시 크기 넣어도 (`int[size] nums`)
- 스펠링/대소문자 실수 (문맥에서 모호하지 않으면)
- `()` 누락 (파라미터 없는 메서드 호출 시)
- `if`, `for`, `while` 조건 주위 `()` 누락

### 4-3. 감점하는 것 (1-Point Penalty) — 진짜 실수

- **v)** `[]` vs `.get()` 혼동 (array와 ArrayList 접근 방식 혼용)
- **w)** 불필요한 side-effect (출력문 추가, precondition 체크 잘못)
- **x)** 지역변수를 사용하되 선언을 하나도 안 함
- **y)** persistent data 파괴 (파라미터로 전달된 값 변경)
- **z)** void 메서드에서 값 return, 또는 생성자에서 return

**감점은 문항당 최대 3점까지만.**

### 4-4. 출제자가 정말 보는 것 (채점 기준 패턴 분석)

2025 FRQ와 CED 샘플에서 반복되는 채점 포인트:

#### Q1 (Methods + Control Structures)
1. 올바른 메서드 호출 (파라미터 타입/개수)
2. 반복 종료 조건의 정확성
3. 누적/카운트 로직
4. **전체 알고리즘 조합** (algorithm point)
5. String 메서드 사용 (substring, indexOf, split 등)

#### Q2 (Class Design)
1. `class` 헤더 선언
2. `private` 인스턴스 변수 (적절한 타입)
3. 생성자 헤더 (파라미터 타입 일치)
4. 생성자에서 인스턴스 변수 초기화
5. 메서드 헤더 (`public`, 반환 타입, 이름)
6. 메서드 내부 로직 (조건 분기 + 상태 업데이트)
7. 적절한 String 또는 값 return

**Q2 핵심:** 클래스를 "처음부터 끝까지" 작성해야 한다. 제공된 표/설명을 코드로 번역하는 능력.

#### Q3 (ArrayList)
1. ArrayList 전체 순회
2. 조건에 맞는 요소 필터링 (getter 메서드 호출)
3. 누적/평균/카운트 계산
4. 올바른 return

#### Q4 (2D Array)
1. 2D 배열 순회 (중첩 루프)
2. 행/열 단위 처리
3. 조건 비교 + 최소/최대/카운트
4. **행-우선 vs 열-우선 순회를 구분**

## 5. Java Quick Reference에서 드러나는 출제 범위

시험 중 제공되는 Reference Sheet = **시험에 나올 수 있는 API의 전체 목록**:

### 반드시 알아야 하는 클래스/메서드
```
String:     length(), substring(from, to), substring(from), indexOf(str), 
            equals(Object), compareTo(String), split(String)
Integer:    MIN_VALUE, MAX_VALUE, parseInt(String)
Double:     parseDouble(String)
Math:       abs(int), abs(double), pow(base, exp), sqrt(x), random()
ArrayList:  size(), add(obj), add(index, obj), get(index), set(index, obj), remove(index)
File:       File(String pathname)          ← 2025-26 신규!
Scanner:    Scanner(File), nextInt(), nextDouble(), nextBoolean(), 
            nextLine(), next(), hasNext(), close()    ← 2025-26 신규!
Object:     equals(Object), toString()
```

**2025-26 변경점:** `File`과 `Scanner` 클래스가 추가됨. 파일 입출력이 시험 범위에 포함.

### MC에서 출제되는 추가 토픽 (FRQ에는 안 나옴)
- **재귀 (Recursion)** — CED Topic 4.16-4.17, MCQ에서만 출제 (CED p.94: "Recursion is assessed only through the multiple-choice section")
- **탐색 알고리즘** — CED Topic 4.14: linear search, binary search (정렬 전제 조건 포함)
- **정렬 알고리즘** — CED Topic 4.15: selection sort, insertion sort, merge sort (과정 tracing)

### 시험에 나오지 않는 것 (출제 범위 밖)
- HashMap, LinkedList, Set 등 다른 컬렉션
- try-catch (예외 처리)
- interface, abstract class (개념은 나오지만 구현 요구 안 함)
- GUI, 그래픽스
- 키보드 입력 (System.in) — CED p.106 EXCLUSION: 파일 입력만 시험 범위
- nextLine()과 다른 Scanner 메서드 혼용 분석 — CED p.106 EXCLUSION
- 고급 알고리즘 (정렬/탐색은 개념+tracing, FRQ에서 직접 구현 요구 안 함)
- generics 심화 (`<? extends T>` 등)

## 6. 출제자의 난이도 설계 원칙

CED와 FRQ 패턴에서 추출한 난이도 조절 방식:

### MC 난이도 스펙트럼
- **쉬움**: 코드 1-3줄의 출력 추적, 기본 문법
- **중간**: 루프 + 조건 조합 추적, 메서드 호출 체인
- **어려움**: 중첩 루프 + 복합 조건, 배열/ArrayList 경계 조건, 재귀 트레이싱

### FRQ 난이도 조절
- **쉬운 점수**: 메서드 헤더 작성, 변수 선언, 단순 getter 호출 → 1점씩 개별 평가
- **어려운 점수**: algorithm point = 전체 로직 통합 → 부분 점수 없음
- **항상 포함**: 경계 조건 (빈 배열, 첫/마지막 요소, 조건에 맞는 요소 없음)

## 7. 4점(Well Qualified) 달성을 위한 시사점

### MC (55% 비중)
- 42문항 중 **25-28문항** 정도 맞추면 4점권
- **코드 추적(trace) 능력**이 핵심 (Practice 3 = 37-53%)
- Unit 2(조건/반복) + Unit 4(컬렉션) 집중

### FRQ (45% 비중)
- 25점 만점에서 **13-16점** 정도 목표
- **부분 점수 전략**: algorithm point를 못 받더라도 non-algorithm point는 개별 획득 가능
- Q2(Class Design)는 구조만 알면 최소 4-5점 확보 가능 (가장 예측 가능)
- Q3(ArrayList)는 for-each 루프 + if + 누적만 하면 3-4점 확보
- Q1과 Q4는 로직이 더 복잡하지만, 메서드 호출/루프 작성만으로도 부분 점수

### 감점 피하는 법
- `[]` vs `.get()` 절대 혼용하지 않기
- `System.out.println()` 불필요하게 추가하지 않기  
- 파라미터로 받은 데이터를 변경하지 않기
- void 메서드에서 return value 쓰지 않기

## 9. 출제자가 중시하는 역량 요약 (우선순위)

1. **코드 추적 (Trace)** — 주어진 코드의 실행 결과를 정확히 예측
2. **루프 + 조건 조합** — for, while, for-each + if/else 중첩
3. **메서드 호출** — 올바른 객체에 올바른 파라미터로 호출
4. **ArrayList 조작** — 순회, 추가, 제거, 조건 필터링
5. **클래스 설계** — private 변수, 생성자, getter, 비즈니스 메서드
6. **2D 배열** — 행/열 순회, 경계 조건
7. **String 메서드** — substring, indexOf, equals, compareTo, split
8. **산술/논리 표현** — 정수 나눗셈, 형변환, 복합 boolean 조건
