### 쉬움 (1~5번) — 변수, 출력, 산술연산

**B1.** 다음 코드의 출력 결과는?
```java
int x = 10;
int y = 3;
System.out.println(x / y);
```
- (A) 3.33
- (B) 3
- (C) 4
- (D) 3.0

> **정답: (B)** — int끼리의 나눗셈은 소수점 이하를 버린다.

---

**B2.** 다음 중 올바른 변수 선언은?
```java
// I
int 2ndPlace = 2;
// II
double price = 9.99;
// III
String name = "Kim";
// IV
boolean isReady = "true";
```
- (A) II만
- (B) II, III만
- (C) I, II, III만
- (D) II, III, IV만

> **정답: (B)** — I은 숫자로 시작하는 변수명(에러), IV는 boolean에 String을 대입(에러).

---

**B3.** 다음 코드의 출력 결과는?
```java
int a = 5;
int b = a;
a = 10;
System.out.println(b);
```
- (A) 5
- (B) 10
- (C) 15
- (D) 컴파일 에러

> **정답: (A)** — primitive 타입은 값을 복사한다. b에 5가 복사된 후 a를 변경해도 b는 그대로 5.

---

**B4.** 다음 코드의 출력 결과는?
```java
System.out.println("AP" + "CSA");
System.out.println(3 + 4 + "AP");
System.out.println("AP" + 3 + 4);
```
- (A) APCSA / 7AP / AP34
- (B) APCSA / 34AP / AP34
- (C) AP CSA / 7AP / AP7
- (D) APCSA / 7AP / AP7

> **정답: (A)** — 문자열 연결 시 왼쪽에서 오른쪽으로 평가. `3+4+"AP"` → 7+"AP" → "7AP". `"AP"+3+4` → "AP3"+4 → "AP34".

---

**B5.** 다음 코드의 출력 결과는?
```java
double d = 5 / 2;
System.out.println(d);
```
- (A) 2.5
- (B) 2.0
- (C) 2
- (D) 컴파일 에러

> **정답: (B)** — `5 / 2`는 int 나눗셈이므로 결과가 2이고, 이것이 double에 저장되어 2.0이 된다.

### 보통 (6~10번) — if/else, for 루프, String 메서드

**B6.** 다음 코드의 출력 결과는?
```java
int score = 85;
if (score >= 90) {
    System.out.println("A");
} else if (score >= 80) {
    System.out.println("B");
} else if (score >= 70) {
    System.out.println("C");
} else {
    System.out.println("F");
}
```
- (A) A
- (B) B
- (C) B와 C 모두
- (D) C

> **정답: (B)** — 85는 `>= 90` 거짓, `>= 80` 참이므로 "B"를 출력하고 나머지는 건너뛴다.

---

**B7.** 다음 코드의 출력 결과는?
```java
int sum = 0;
for (int i = 1; i <= 5; i++) {
    sum += i;
}
System.out.println(sum);
```
- (A) 5
- (B) 10
- (C) 15
- (D) 20

> **정답: (C)** — 1+2+3+4+5 = 15.

---

**B8.** 다음 코드의 출력 결과는?
```java
String s = "Computer";
System.out.println(s.substring(0, 3));
System.out.println(s.indexOf("put"));
```
- (A) Com / 3
- (B) Comp / 3
- (C) Com / 4
- (D) Comp / 4

> **정답: (A)** — `substring(0, 3)`은 인덱스 0~2까지 = "Com". `indexOf("put")`은 "put"이 시작하는 인덱스 = 3.

---

**B9.** 다음 코드에서 `count`의 최종 값은?
```java
int count = 0;
for (int i = 0; i < 10; i++) {
    if (i % 3 == 0) {
        count++;
    }
}
```
- (A) 3
- (B) 4
- (C) 5
- (D) 10

> **정답: (B)** — i = 0, 3, 6, 9일 때 3의 배수. 총 4회.

---

**B10.** 다음 코드의 출력 결과는?
```java
String word = "Hello";
String upper = word.toUpperCase();
System.out.println(word);
System.out.println(word.equals(upper));
```
- (A) HELLO / true
- (B) Hello / false
- (C) Hello / true
- (D) HELLO / false

> **정답: (B)** — String은 immutable이므로 `toUpperCase()`는 새 객체를 반환하고 원본 `word`는 변경되지 않는다. "Hello"와 "HELLO"는 다르므로 `equals`는 false.

### 어려움 (11~15번) — 배열, ArrayList, 클래스 설계

**B11.** 다음 코드의 출력 결과는?
```java
int[] arr = {5, 10, 15, 20, 25};
System.out.println(arr[2]);
System.out.println(arr.length);
```
- (A) 10 / 5
- (B) 15 / 5
- (C) 15 / 4
- (D) 10 / 4

> **정답: (B)** — 인덱스 2는 세 번째 원소 = 15. `length`는 배열 크기 = 5.

---

**B12.** 다음 코드의 출력 결과는?
```java
import java.util.ArrayList;

ArrayList<String> list = new ArrayList<String>();
list.add("A");
list.add("B");
list.add("C");
list.add(1, "X");
list.remove(2);
System.out.println(list);
```
- (A) [A, X, C]
- (B) [A, X, B]
- (C) [A, B, C]
- (D) [X, A, C]

> **정답: (A)** — add 후: [A, B, C]. `add(1, "X")` → [A, X, B, C]. `remove(2)` → 인덱스 2의 "B" 제거 → [A, X, C].

**B13.** 다음 클래스가 주어졌을 때, 출력 결과는?
```java
public class Dog {
    private String name;
    private int age;

    public Dog(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() { return name; }
    public int getAge() { return age; }
}

// main에서:
Dog d1 = new Dog("Max", 3);
Dog d2 = new Dog("Buddy", 5);
System.out.println(d1.getName() + " " + d2.getAge());
```
- (A) Max 3
- (B) Buddy 5
- (C) Max 5
- (D) Buddy 3

> **정답: (C)** — `d1.getName()` = "Max", `d2.getAge()` = 5. 결합하면 "Max 5".

---

**B14.** 다음 코드에서 컴파일 에러가 발생하는 줄은?
```java
public class Student {
    private String name;
    private int grade;

    public Student(String n, int g) {
        name = n;
        grade = g;
    }

    public int getGrade() { return grade; }
}

// main에서:
Student s = new Student("Kim", 11);     // 줄 1
System.out.println(s.name);              // 줄 2
System.out.println(s.getGrade());        // 줄 3
```
- (A) 줄 1
- (B) 줄 2
- (C) 줄 3
- (D) 에러 없음

> **정답: (B)** — `name`은 `private`이므로 클래스 외부에서 직접 접근할 수 없다. getter 메서드를 통해 접근해야 한다.

**B15.** 다음 메서드가 하는 일을 가장 잘 설명한 것은?
```java
public static int mystery(ArrayList<Integer> list) {
    int result = list.get(0);
    for (int i = 1; i < list.size(); i++) {
        if (list.get(i) > result) {
            result = list.get(i);
        }
    }
    return result;
}
```
- (A) 리스트의 합계를 반환한다
- (B) 리스트의 최솟값을 반환한다
- (C) 리스트의 최댓값을 반환한다
- (D) 리스트의 마지막 원소를 반환한다

> **정답: (C)** — `result`를 첫 원소로 초기화한 뒤, 더 큰 값을 발견할 때마다 갱신한다. 최종적으로 리스트의 최댓값을 반환한다.

## 교사용 수업 조정 가이드

### Level 0: 완전 초보 (Section B 0~3점)

**특징**: 프로그래밍 경험이 없거나 변수 개념조차 낯선 학생

**수업 계획 조정**:
- 기본 10회 과정을 **그대로** 진행
- 1~2회차에서 시간을 충분히 투자 (변수, 타입, 출력 반복 연습)
- 매 수업 시작 10분을 전 수업 복습에 할당
- 코드를 직접 타이핑하는 연습 병행 (IDE 사용법부터)
- FRQ는 8회차 이후부터 도입

**주의사항**:
- 추상적 설명보다 구체적 비유 사용 (변수 = 상자, 메서드 = 레시피)
- 한 수업에 새 개념 2개 이하
- 숙제는 짧고 명확하게 (코드 5줄 이하)

---

### Level 1: 약간의 경험 (Section B 4~8점)

**특징**: 기본 문법은 알지만 객체지향이나 자료구조가 약한 학생

**수업 계획 조정**:
- 1~2회차를 **1회로 압축** (변수, 연산자, 조건문을 빠르게 리뷰)
- 절약한 시간을 Unit 5(Writing Classes), Unit 6(Array)에 투자
- 3회차부터 본격 시작
- FRQ를 6회차부터 조기 도입

**주의사항**:
- 학생이 "아는 부분"도 Java 문법으로 정확히 쓸 수 있는지 확인
- Python 경험자는 세미콜론, 중괄호, 타입 선언 등의 차이에 주의
- "대충 알겠다"는 느낌과 시험에서 맞추는 것은 다름을 인지시킬 것

---

### Level 2: 다른 언어 경험 풍부 (Section B 9~12점)

**특징**: 프로그래밍 자체는 익숙하나 Java 특유의 문법/패턴이 부족한 학생

**수업 계획 조정**:
- 1~3회차를 **1~2회로 압축** (차이점 위주로 빠르게)
- Java 고유 개념에 집중: `equals()` vs `==`, 참조 타입, ArrayList/2D 배열 알고리즘과 FRQ 실전 연습
- **FRQ 집중** — 5회차부터 매 수업 FRQ 1문제 풀기
- 모의시험을 7~8회차에 1회 추가

**주의사항**:
- Python에서 오는 학생: 타입 시스템, 접근 제어자, 클래스 설계 문법 차이 강조
- 알고리즘은 알아도 Java 방식으로 구현하는 연습 필요
- AP 시험 특유의 문제 스타일(trace 문제, 코드 완성)에 익숙해지도록

---

### Level 3: 대부분 이미 숙지 (Section B 13~15점)

**특징**: Java와 OOP 개념을 이미 잘 알고 있는 학생

**수업 계획 조정**:
- 6회차(배열)부터 시작
- **모의시험 + FRQ 집중 모드**
- 매 수업: 개념 리뷰 20분 + FRQ 실전 연습 40분
- 최소 3회의 풀 모의시험(MCQ 40문제 + FRQ 4문제) 실시
- 약점 토픽만 골라서 보강

**주의사항**:
- 자만하지 않도록 초반에 어려운 FRQ를 줘서 현실 감각 잡아주기
- 시간 관리 연습 중요 (MCQ: 1문제 평균 1.5분, FRQ: 1문제 평균 22분)
- 4점/5점 목표를 명확히 설정하고 역산
